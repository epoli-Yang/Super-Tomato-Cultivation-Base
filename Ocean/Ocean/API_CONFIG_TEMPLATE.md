## 3. 🚀 前端业务逻辑与数据对接指引

基于当前的表结构和工作流设计，前端页面的核心数据流转逻辑如下：

### 3.1 “首页” (小白助手 Agent)
- **数据来源**：直接调用 **Coze Chat 工作流**。
- **逻辑**：用户输入问题 -> 调用 Chat API -> 获取流式或非流式响应展示在聊天界面。

### 3.2 “行程” 页面
- **触发规划**：用户点击“自动规划”按钮 -> 前端静默调用 **Coze Chat 工作流**，传入固定 prompt `"帮我规划在海洋馆的行程"`。
- **页面渲染**：从 Supabase 数据库的 `asd_details` 表中，读取该用户的 `pre_trip` 字段。
- **数据提取**：解析 `pre_trip` JSON 数据，遍历其中的 `itinerary` 数组（包含 `id`, `name`, `note`），渲染出行程列表和场馆信息。

### 3.3 “我的” 页面 (User Center)
- **用户卡片**：从 Supabase 的 `profiles` 表中读取对应用户的基本信息（如 `nickname`, `role` 等）进行展示。
- **个人档案入口**：点击进入档案详情，请求 `asd_details` 表获取该用户的详细感官数据。
- **雷达图渲染**：读取 `asd_details` 表中的 `sensory_object` 字段（如 `{"v":0.8, "a":0.5...}`），将其转换为 ECharts/uCharts 需要的格式，渲染出五感敏感度雷达图。
